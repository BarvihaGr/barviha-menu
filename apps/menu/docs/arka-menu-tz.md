# ТЗ: диджитал-меню «Арка» (кальяны / бар / кухня)

Слияние двух референсов + собственный контент:
- **Timeless (timeless.club/menu/bar, /menu/aromaty)** → визуальный язык, структура страницы, компоненты, микро-детали, сетка/отступы, типографическая шкала.
- **Киевская (наша)** → инженерная система: как цвет/шрифт/lux-вариант заведены через CSS-переменные и переключаются по slug локации; этот же механизм расширяем на новые города.
- **Арка** → свой цвет (уже определён в `arka-lab`), свои блюда и цены (уже в `menu-generated.ts`), своя витрина.

---

## 1. Что берём у Timeless (design/структура/компоненты/отступы/типографика)

### 1.1 Сетка
Fluid-grid через `calc()` от вьюпорта, не фиксированный Tailwind-контейнер:
```
--grid-columns: 12 (desktop) → 8 (mobile/tablet)
--offset-x: 30px → 20px → 16px
--header-height: 102px → 100px → 80px → 70px
--grid-gap: 0
```
Брейкпоинты плотные, «по контенту»: 767 / 900(landscape) / 1024(portrait) / 1199 / 1280 / 1366 / 1439 / 1440 / 1600+ — не 3-4 стандартных, а много промежуточных, чтобы на любой ширине сетка не «прыгала».

### 1.2 Форма компонентов
Только два состояния радиуса — среднего (8-12px) не существует:
- **Острые прямоугольники (`radius: 0`)** — фотокарточки блюд/ароматов, карточки категорий.
- **Полная пилюля (`radius: 50px/100%`)** — табы категорий, чипы фильтров, переключатель локации/языка, круглые иконки.

### 1.3 Состояния через CSS-переменные, не классы
Один компонент описывает default/hover/active через переменные:
```
--color-default / --color-hover / --color-active
--bg-color-default / --bg-color-hover / --bg-color-active
--border-color-default / --border-color-hover / --border-color-active
```
Смена состояния = смена значения переменной, а не смена className. Переносим этот паттерн 1:1 в `--cm-*` токены Арки.

### 1.4 Переходы
- Общие: `.25s–.4s ease` / `cubic-bezier(.25,.25,0,1)` на `background-color`, `border-color`, `color`.
- Смена категории/раздела: `clip-path 1s cubic-bezier(.25,.25,0,1)` — «раскрытие» контента, не fade по opacity. Это самая узнаваемая «дорогая» деталь Timeless — переносим на смену вкладки категории (кальяны/бар/кухня) и на смену секции внутри категории.
- `box-shadow` почти не используется (плоский дизайн); единственное исключение — тонкий glow `0 0 14px 5px` на активной точке/кнопке.
- Нет `backdrop-filter`/glassmorphism на модалках Timeless — но у нас в `CoffeeHeader` уже есть `backdrop-blur-md`, оставляем как есть (это наше улучшение, не откатываем).

### 1.5 Типографическая шкала (не сам шрифт — шрифт см. п.2)
- Заголовки категорий — **caps**, шкала 24/34/38/42/48/50px под десктоп, крупнее (70-100px) для геро-заголовков локации.
- Мелкие лейблы (подписи разделов, "1/4"-счётчики, город/язык) — caps, `letter-spacing: 0.05–0.07em` (широкий трекинг).
- Заголовки — `letter-spacing: -0.007…-0.025em` (слегка сжато).
- Цена — моноширинно выровнена по правому краю карточки/строки.

### 1.6 Структура страницы «Ароматы» / «Бар» → переносим на кальяны/бар/кухню Арки
1. Sticky header: лого + локация/город + язык + бургер.
2. Табы верхнего уровня: **Кальяны / Бар / Кухня** (у Timeless — Ароматы/Бар/Закуски), с индикатором позиции (аналог их "1/4").
3. Внутри категории — под-разделы (у Timeless: Классика/Авторские/Фруктовые; у нас: sub-секции по `item.sub`, уже реализовано в `CoffeeMenuList`) со sticky scroll-spy чипами.
4. Карточка позиции: фото (не всегда есть — ок, у части позиций Арки фото ещё нет), название caps, короткое описание, цена справа.
5. Внизу — сноска мелким текстом («количество позиций и цены могут отличаться») + контакты/соцсети — уже есть в `CoffeeLuxContacts`, не трогаем.

## 2. Что берём у Киевской (цвет-система / шрифт-система / city-адаптация)

Это не «взять цвет Киевской себе» — Арка уже имеет свой цвет (см. п.3). Берём **механизм**:

### 2.1 Токен-система (`apps/menu/src/lib/coffee-design.ts`)
- `COFFEE_DESIGN_SLUGS` — сет локаций на новом дизайне. **Добавить `arka`.**
- `COFFEE_PALETTE: Record<slug, CoffeePalette>` — своя палитра `--cm-bg/--cm-surface/--cm-text/--cm-border/...` на локацию. **Добавить `ARKA_PALETTE`.**
- `COFFEE_ACCENT: Record<slug, hex>` — прямой акцент локации, иначе — цвет ветки метро. У Арки уже задано `arka → cyan (Филёвская)` в `location-theme.ts`, но нужен свой брендовый акцент `#C49262` (уже используется как «цветокор Арки» — им же перекрашен Домодедово). **Добавить `COFFEE_ACCENT.arka = '#C49262'`.**
- `coffeeAccentStyle(slug)` — инлайн-стиль на корень поддерева, прокидывает палитру + шрифтовые переменные. Работает для Арки без изменений кода компонента, только через добавление данных.

### 2.2 Шрифт-система
- `--cm-font-display` / `--cm-font-body` — базовые (Raleway/Jost/Tenor Sans + Manrope).
- Lux-локации (`LUX_HOME_SLUGS`: Ереван, Киевская) переопределяют display на `Cormorant SC` — это и есть «шрифт от Киевской», который берём для Арки: **добавить `arka` в `LUX_HOME_SLUGS`.** Итог: заголовки Арки — Cormorant SC (элегантный лаунж-серотип), текст — Manrope. Совпадает с духом «дорогого минимализма» Timeless без покупки их закрытых шрифтов TT-Ricordi-Allegria/TT Drugs.

### 2.3 City-адаптация (масштабирование на много городов)
Механизм уже существует и рассчитан на рост сети:
- `getMetroColor(slug)` — точки в Москве/СПб получают цвет своей ветки метро (визуальный маркер в переключателе локаций); точки вне МСК/СПб (другие города — Ереван, Ташкент, Пенза и т.д.) — золото по умолчанию.
- `COFFEE_ACCENT` / `COFFEE_PALETTE` — точечное переопределение акцента/палитры конкретной локации поверх дефолта, ключ — slug. Для новой точки в другом городе — просто новая запись в обеих картах, без изменения компонентов.
- `effectiveSlug()` в `client.ts` (`packages/db`) — если у локации ещё нет своего покрытия меню/цен (`COVERED_SLUGS`), она временно показывает меню `baumanskaia` (сетевая база). У Арки уже есть собственное покрытие (`prices.arka` в `menu-generated.ts`), поэтому фолбэк не задействуется.
- Middleware (`apps/menu/src/middleware.ts`) сейчас редиректит все локации, кроме `kievskaia`/`arka`/служебных, на Киевскую — это временный «мягкий запуск» только двух точек. Когда добавляются новые города — просто расширяется `passThrough`.

**Вывод: инфраструктура на много городов уже готова, ничего архитектурно менять не нужно — только регистрировать новые slug'и в тех же 3-4 картах (palette/accent/lux/metro).**

## 3. Свой цвет Арки (не трогаем Киевскую)

Уже определён в `arka-lab/page.tsx` как изолированная песочница и используется как «цветокор Арки» (см. комментарий в `BRONZE_PALETTE` — этим же цветом перекрашен Домодедово):

```
BG      #1B110A   — почти чёрный тёплый (не нейтральный чёрный Timeless — с бронзовым подтоном)
SURFACE #241710   — поверхности карточек/чипов
ACCENT  #C49262   — бронза-золото (единственный цветной акцент, как у Timeless — один акцент, не палитра)
TEXT    #F4E9D5   — тёплый кремовый текст
```

Оформляется в `coffee-design.ts` как `ARKA_PALETTE`, аналогично `BRONZE_PALETTE`/`KIEV_PALETTE`:
```ts
const ARKA_PALETTE: CoffeePalette = {
  '--cm-bg': '#1B110A',
  '--cm-surface': '#241710',
  '--cm-surface-2': '#2E2015', // на тон светлее surface — для вложенных карточек/hover
  '--cm-text': '#F4E9D5',
  '--cm-text-soft': '#E8D4B0',
  '--cm-muted': 'rgba(244, 233, 213, 0.60)',
  '--cm-muted-dim': 'rgba(244, 233, 213, 0.40)',
  '--cm-border': 'rgba(196, 146, 98, 0.20)',
  '--cm-logo-invert': '1', // тёмный фон — лого белым
  '--cm-photo': 'contrast(1.05) saturate(1.02)',
  '--cm-photo-veil': 'transparent',
  '--cm-accent-text': '#1B110A', // текст поверх залитых золотом кнопок/пилюль
};
```

### 3.1 Sharp-corner карточки (деталь Timeless, без вреда для Киевской)
Сейчас радиус фото/карточек в `CoffeeItemCard` захардкожен в JSX (`rounded-2xl`), общий для всех coffee-локаций. Чтобы у Арки был острый Timeless-угол, а Киевская осталась как есть — вводим переменную:
```
--cm-card-radius: 16px;   /* дефолт — как сейчас, Киевская/Ереван/Домодедово не меняются */
```
и переопределяем в `ARKA_PALETTE`:
```
--cm-card-radius: 0px;
```
`CoffeeItemCard` переключается с `rounded-2xl` на `rounded-[var(--cm-card-radius)]`. Пилюли (табы/чипы) везде остаются `rounded-full` — не трогаем.
